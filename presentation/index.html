📜 History of the Internet

  • 1969

    ARPANET — first packet-switched network, connecting 4 US universities. The first message sent was "lo" (login crashed after two letters).

  • 1983

    TCP/IP adopted — ARPANET switches to the TCP/IP protocol suite, creating the foundation of the modern internet.

  • 1991

    World Wide Web born — Tim Berners-Lee publishes the first website at CERN (info.cern.ch), introducing HTTP and HTML.

  • 1993

    Mosaic browser — first graphical web browser, making the web accessible to the general public.

  • 2000s

    Web 2.0 — rise of social media, user-generated content, broadband, and JavaScript-powered web apps.

📦 How a Packet Travels

Example: You type https://google.com in your browser
  • 🔍 DNS lookup — resolves "google.com" to an IP address (142.250.x.x)
  • 🤝 TCP handshake — SYN → SYN-ACK → ACK (3 packets)
  • 🔒 TLS handshake — Client Hello, Server Hello, certificates exchanged
  • 📨 HTTP GET request — browser requests the page
  • 📥 HTTP response — server sends HTML, CSS, JavaScript
  • 🖥️ Rendering — browser builds the DOM and displays the page

🔄 TCP vs UDP

TCP — Reliable

  • Connection-oriented (handshake)
  • Guarantees delivery and order
  • Error checking and retransmission
  • Slower — more overhead
  • Used for: HTTP, email, file transfer

UDP — Fast

  • Connectionless — no handshake
  • No delivery guarantee
  • No retransmission
  • Faster — minimal overhead
  • Used for: video calls, gaming, DNS, QUIC
Modern trend: QUIC (used by YouTube, Google) runs over UDP but adds its own reliability layer, combining the speed of UDP with the safety of TCP.

🔐 HTTP vs HTTPS

🔓 HTTP

  • Port 80
  • All data sent as plain text
  • Anyone on the network can read it
  • No certificate required
  • Wireshark shows full content
  • Used in early 1990s web

🔒 HTTPS

  • Port 443
  • All data encrypted with TLS
  • Content invisible to sniffers
  • Requires SSL/TLS certificate
  • Only IP + SNI visible in Wireshark
  • Mandatory for all modern sites

🔬 Network Analysis Tools

🦈 Wireshark

The most popular packet analyser. Captures all packets on a network interface using libpcap/WinPcap. Shows full protocol stack (OSI layers) for each packet. Supports display filters like http, tls, tcp.port==80.

📊 Protocol Hierarchy

Wireshark's Statistics tool shows which protocols dominate traffic. In a real capture: QUIC = 49.8%, TLS = 8.2%, DNS = 4.8%, plain HTTP = only 0.1% — proving the modern web is almost entirely encrypted.

📡 tcpdump

Command-line packet capture tool for Linux/macOS. Saves traffic to .pcap files for later analysis in Wireshark. Example: tcpdump -i eth0 port 80 captures all HTTP traffic.

🔍 nmap

Network scanner that discovers hosts and open ports. Used in security auditing to map network topology. Example: nmap -sV 192.168.0.0/24 scans an entire local network.

💻 Web Technologies Stack

HTML — Structure

HyperText Markup Language defines the structure and content of web pages. The first version was created by Tim Berners-Lee in 1991. Current version: HTML5.

CSS — Style

Cascading Style Sheets control visual presentation — colours, fonts, layout, animations. Introduced in 1996. Current version: CSS3 with variables, grid, and flexbox.

JavaScript — Behaviour

The programming language of the web, running in the browser. Powers interactive UIs, API calls, and real-time updates. Node.js also runs it on servers.

Reveal.js — This Presentation

A JavaScript framework for creating HTML presentations. Supports Markdown, themes, syntax highlighting, and both horizontal and vertical navigation. Free and open-source.

✅ Summary

  • 🌐 The internet began with ARPANET (1969) and the Web was born at CERN in 1991
  • 🔢 The OSI 7-layer model describes how data travels from application to physical wire
  • ⚙️ Key protocols: HTTP, HTTPS, TCP, UDP, DNS, QUIC — each with a specific role
  • 🔓 HTTP is fully readable in Wireshark — no privacy whatsoever
  • 🔒 HTTPS/TLS encrypts all content — only IP and SNI hostname are visible
  • 🦈 Wireshark proves: modern traffic is 99.9% encrypted (only 0.1% plain HTTP)
  • 💻 The web is built on HTML + CSS + JavaScript, deployed via GitHub Pages
